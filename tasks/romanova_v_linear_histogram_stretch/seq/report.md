## 1. Контекст

Линейное растяжение гистограммы — базовый метод повышения контрастности изображения, преобразующий пиксели так, чтобы
выходные значения заполняли весь диапазон [0, 255].

Последовательная версия служит эталоном корректности и базой (baseline) для измерения ускорения и эффективности
параллельных реализаций.

## 2. Постановка задачи

**Входные данные:** одномерный массив `uint8_t` пикселей изображения (значения в диапазоне 0–255).

**Выходные данные:** массив того же размера после линейного растяжения гистограммы. Минимальное значение выходного
массива — 0, максимальное — 255.

Растяжение происходит поэлементно в соответствии с формулой $out[i] = \frac{(in[i] - min_v)*255}{max_v-min_v}$.

**Ограничения:** входной массив не пуст.

## 3. Базовый алгоритм

На вход подаётся массив in[0..n-1]

```text
1. min_v = min(in), max_v = max(in)
2. Если min_v == max_v: out ← in, вернуть out
3. diff = max_v - min_v
4. Для i от 0 до n-1:
        out[i] = (in[i] - min_v) * 255 / diff
5. Вернуть out
```

**Асимптотика по времени:** O(n) — один проход для поиска min/max + один проход для преобразования.

**Асимптотика по памяти:** O(n) — входной и выходной массивы.

**Инварианты:** `0 ≤ out[i] ≤ 255` для всех i.

**Критерий корректности:** В выходном массиве минимум совпадает с максимумом или минимум = 0, максимум = 255.

## 4. Детали реализации

В рамках архитектуры `BaseTask` используются стандартные методы `ValidationImpl`, `PreProcessingImpl` и
`PostProcessingImpl` для формальной проверки, подготовки и постобработки векторов. Основная логика линейного растяжения
гистограммы сосредоточена в методе `RunImpl`:

- **ValidationImpl:** проверяет корректность входных данных. Возвращает `false`, если входной массив пуст, и `true` в
  противном случае.

- **PreProcessingImpl:** подготавливает выходной массив. Выделяет память размером, равным входному массиву, с помощью
  `GetOutput().resize()`.

- **RunImpl:** метод, организующий основное вычисление. Он извлекает исходный массив из `GetInput()`, выполняет поиск
  минимального и максимального значений, затем применяет линейное преобразование к каждому пикселю и сохраняет результат
  в `GetOutput()`.

- **PostProcessingImpl:** завершающая обработка. В последовательной версии не требует действий (всегда возвращает
  `true`).

**Алгоритм поиска min/max:** В последовательной версии используется `std::ranges::minmax_element` для однопроходного
поиска минимального и максимального элементов массива. Этот алгоритм эффективнее двух отдельных вызовов
`std::min_element` и `std::max_element`, так как требует только одного прохода по данным.

**Алгоритм преобразования:** После получения глобальных `min_v` и `max_v` проверяется особый случай монохромного
изображения. Если `min_v == max_v`, преобразование пропускается, выходной массив становится копией входного. Иначе
вычисляется разность `diff = max_v - min_v` и для каждого пикселя выполняется целочисленное преобразование:

```cpp
out[i] = (pix - min_v) / diff * 255
```

Порядок операций (сначала деление, затем умножение) выбран для минимизации риска переполнения промежуточных результатов
при использовании 8-битных целых типов.

```cpp
// Файл: seq/src/ops_seq.cpp, метод RunImpl
bool RomanovaVLinHistogramStretchSEQ::RunImpl() {
  const InType &in = GetInput();
  OutType &out = GetOutput();

  auto [min_it, max_it] = std::ranges::minmax_element(in);
  uint8_t min_v = *min_it;
  uint8_t max_v = *max_it;

  if (min_v == max_v) {
    out = in;
    return true;
  }

  const uint8_t diff = max_v - min_v;

  for (size_t i = 0; i < in.size(); i++) {
    uint8_t pix = in[i];
    out[i] = (pix - min_v) / diff * 255;
  }

  return true;
}
```

## 5. Проверка корректности

Корректность последовательной версии проверялась путем проверки выполнения критерия корректности для выходных данных на
расширенном наборе из 6 функциональных тестов:

1. **Изображение 1×1 (один пиксель):** проверка граничного случая минимального размера и ветки `min_v == max_v`.

2. **Изображение 5×5:** проверка правильности растяжения на малом размере.

3. **Изображение 13×13:** проверка работы алгоритма на среднем размере входных данных.

4. **Изображение 70×70:** проверка устойчивости при большем количестве пикселей.

5. **Изображение 200×200:** проверка корректности на значительном объёме данных.

6. **Реальное монохромное изображение:** — проверка ветки `min_v == max_v` на реальных данных, загружаемых из файла.

**Генерация данных:** в первых 5 случаях значения пикселей генерируются по формуле
`in[i][j] = low + ((j * 13 + i * 109) % range)`, где `low = 75`, `range = 50`. Т.е. значения пикселей лежат в диапазоне
[75, 124].

**Критерии корректности:**

Для каждого теста проверяется выполнение одного из двух условий:

- Выходное изображение монохромное (`min == max`) — тогда все пиксели остаются неизменными.
- Выходное изображение имеет полный динамический диапазон: минимальное значение равно 0, максимальное значение
  равно 255.

```cpp
// tests/functional/main.cpp, критерий корректности
bool CheckTestOutputData(OutType &output_data) final {
  auto [min_it, max_it] = std::ranges::minmax_element(output_data);
  return (*min_it == 0 && *max_it == 255) || (*min_it == *max_it);
}
```

ассоциативная операция, а преобразование пикселей может выполняться независимо.
